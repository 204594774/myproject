# 系统升级方案：从“赛事核心”向“全生命周期项目管理”扩展

根据DeepSeek的分析与您的需求，我们将在现有“赛事”功能基础上，通过**最小改动**实现**最大收益**，将系统升级为符合论文题目的“大学生创新创业项目管理系统”。

## 核心思路
利用现有的 `competitions`（赛事/批次）表和 `form_config`（动态配置）字段，将“赛事”概念泛化为“申报批次”。
- **旧逻辑**：赛事 -> 报名 -> 项目
- **新逻辑**：申报通道（可以是年度大创计划，也可以是挑战杯赛事） -> 按照限定类型申报 -> 项目

## 实施步骤

### 1. 前端界面重构 (UI/UX)
- **概念重命名**：
    - 将“赛事管理”更名为 **“申报批次管理”**（Admin）。
    - 将“赛事列表”更名为 **“项目申报大厅”**（Student）。
- **视觉区分**：
    - 在申报大厅中，增加**“分类标签”**，区分“常规训练项目”（如大创年审）和“竞赛活动”（如互联网+）。

### 2. 管理员功能增强 (Admin)
- **批次类型定义**：
    - 在创建/编辑“赛事”（现为批次）时，增加 **“批次性质”** 选择（常规项目 / 学科竞赛）。
- **项目类型约束**：
    - 利用 `form_config` 字段，新增 **“允许申报的项目类型”** 配置项（多选框）。
    - *收益*：您可以创建一个名为“2025年创新训练计划”的批次，并勾选只允许申报“创新训练”类项目，从而实现精准管理。

### 3. 学生申报逻辑升级 (Student)
- **动态类型过滤**：
    - 修改项目申请弹窗，使 **“项目类型”** 下拉菜单根据当前批次的配置动态显示。
    - 如果批次只允许一种类型（如仅限“创业实践”），系统将自动选中并锁定该类型。

### 4. 数据库与后端 (Backend)
- **零Schema变更**：
    - 所有新配置（批次性质、允许类型）均存储在现有的 `form_config` JSON字段中，无需修改数据库结构，降低风险。

## 预期效果
修改后，系统将呈现出清晰的 **“两级架构”**：
1.  **顶层**：项目申报批次（覆盖大创、互联网+、挑战杯等全口径）。
2.  **底层**：具体项目实例（通过批次创建，继承批次属性）。

这完全符合“大学生创新创业项目管理系统”的设计要求，既包含了日常训练项目的全生命周期管理，也兼容了竞赛项目的特殊流程。
